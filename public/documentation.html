This is an application that will let you sort through different genres of movies
and directors, and let you save your favorite movies to your account.

How To Use This API

Business Logic URL HTTP Method Query Parameters Request Body Data Format Response Body Data Format
Returns all current users /users GET N/A N/A A JSON object of all the current users
Returns a specific user by name /users/:userName GET :userName N/A A JSON object of all a specific user
Returns a list of all movies /movies GET N/A N/A A JSON object of all the movies
Returns a specific movie by title /movies/:title GET :title N/A A JSON object of one movie
Allows users to search for a movie by genre /users/genres/:genreName GET :genreName N/A returns all movies with the genre searched
Returns movies by directors /movies/directors/:directorName GET :directorName N/A A JSON object of all the movies with that searched director
Allows a user to create a new account /users/ POST N/A You Must Enter a JSON object of a new user ie.{ "Username": "PizzaLover", "Password": "HungryHowies9898", "Email": "PizzaHunger@gmail.com", "Birthday": "01-07-1993" } *note Username,Password,Email,Birthday must be capitalized and all fields must be field out. A JSON object of a new user added plus an ID number and empty array to store users favorite movies ie. { "Username": "PizzaLover", "Password": "HungryHowies9898", "Email": "PizzaHunger@gmail.com", "Birthday": "1993-01-07T05:00:00.000Z", "FavoriteMovies": [], "_id": "646246127a99d582c7aa8f4d", "__v": 0 }
Saves a users favorite movie /users/:userName/:MovieID POST :userName/:MovieID In the url /:MovieID use the ObjectID of the movie document in the movie collection of a specific movie you like. it will return a JSON Object with the ussers id, username,password email, birthday and the favorite movie you added will be in the array of favorite movies
Allows a user to be deleted /users/:userName DELETE :userName N/A Response text saying ie. This "User" has been deleted
Allows a user to update their user information /user/:userName PUT :userName A JSON object of the four user categories, each categorize ie. Username, Password etc. must be capitlized and filled in even if its not updated. For example { "Username": "JuniorPizzaBoy", "Password": "HungryHowiesPizza9898", "Email": "PizzaHungerandSoda@gmail.com", "Birthday": "October 31, 2005" } A JSON object of the updated information you changed and the information you didn't change. If you updated all categories all information when come back updated ie. { "_id": "646238c8a0d5073f558507c1", "Username": "JuniorPizzaBoy", "Password": "HungryHowiesPizza9898", "Email": "PizzaHungerandSoda@gmail.com", "Birthday": "2005-10-31T05:00:00.000Z", "FavoriteMovies": [], "__v": 0 }
Removes a users favorite movie /users/:userName/movies/:MovieID DELETE :userName/:MovieID In the url /:MovieID use the ObjectID of the movie document in the movie collection of a specific returns a JSON object of all users information and updated FavoriteMovies missing the movie you requested to be deleted.
